### 1. Identify Data Model Objects

In SmartDocAI, DynamoDB is applied for 3 main components:

##### Table 1: User Profile Management Table (`smartdocai-user-profiles`)
- **Partition Key (PK): `user_id` (String) - Corresponds to `sub` (UUID) generated by AWS Cognito.**
- **Attributes:**
  - `email` (String): Email address.
  - `fullname` (String): Full name.
  - `phone` (String): Phone number (E.164 standard: `+84...`).
  - `dob` (String): Date of birth (`YYYY-MM-DD`).
  - `avatar` (String, Optional): Base64 string (if image size < 300KB).
  - `avatar_url` (String, Optional): S3 URL path (if image size > 300KB).
  - `subscription_plan` (String): Service tier (`free` / `pro`).
  - `document_quota` (Number): Document limit (Default: 50).
  - `documents_used` (Number): Uploaded document count.
  - `user_preferences` (Map): Language & theme settings (`{ language: "vi", theme: "dark" }`).
  - `created_at` (Number): Creation timestamp (Unix timestamp).
  - `updated_at` (Number): Last updated timestamp.

##### Table 2: SmartDoc-Documents Table
- **Partition Key (PK): `user_id` (String) - Ensures document authorization per user (Multi-tenancy).**
- **Sort Key (SK): `doc_id` (String) - Document UUID identifier.**
- **Attributes: `filename`, `s3_key`, `page_count`, `chunk_count`, `file_size`, `status` (`PROCESSED`/`FAILED`), `created_at`.**

##### Table 3: SmartDoc-ChatSessions Table
- **Partition Key (PK): `user_id` (String).**
- **Sort Key (SK): `timestamp` (Number) or `message_id` (String).**
- **Attributes: `role` (`user`/`assistant`), `message`, `sources` (JSON List), `created_at`.**
